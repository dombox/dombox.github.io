Data Classifications

User Data is classified into three categories. (1) Green Data (2) Yellow Data (3) Red Data

Data Sensitivity

Green Data Low
Yellow Data Medium
Red Data High

Data Access

Consumers Fills their personal data by editing their profile
Businesses Request Consumer Data via Portal
Consumers Give permission to business to access their data via “Teleport”

Green data

If a third party website get hacked, the damage is nearly null in this category. This is because, all data found in this category are Insensitive ones (including email address).

e.g. Back in 2013, 150 million Adobe accounts were hacked. If Adobe had only our green data, they can contact our consumers without any issues, on the other hand, this data is useless in spammers hands.

Because hacking this data is nothing more than crawling Facebook profiles. Note: Everyday websites are getting hacked and user data getting stolen. So this is a major breakthrough.

For most websites, only Green Data is enough. If you are website owner, keep in mind you are discouraging user signups if you request “Yellow Data” and “Green Data” without a sensible reason.

“Green Data” contains the following fields.

First Name, Last Name, Display Name, Preferred Usernames, Domkey, Email, Gender, Avatar, Age Group, Date Joined, Time Zone, Locale, Date Format, Website

Yellow Data

If a third party website that contain the yellow data get hacked, then the damage is minimal.

“Yellow Data” contains the following fields.

Date of Birth, Country, Social Links.

Keep in mind, the consumer has the option to decline Yellow Data and Red Data requests.

Yellow Data and Red Data requires a valid reason for each field.

For instance, Yellow Data contains "Date Of Birth" field. If some website need to access that data, then a valid reason is required from them. e.g. "Adult website. For age verification" is a valid reason. But "To send birthday wishes" is not.

Red Data

Red Data contains highly sensitive fields. Phone Number, Billing Address, Shipping Address

These data will be helpful when signing up for e-commerce websites

Again.. the consumer has the option to decline the Red Data request.

And Red Data requires a valid reason for each field.

A portal that request access for at least one “Red Data” field is called "Red Portal"

A portal that request access for at least one “Yellow Data” data but not “Red Data” fields is called "Yellow Portal"

A portal that request access for only “Green Data” fields is called "Green Portal". By default all portals have full access to this data.

Benefits

Teleport button is designed to attract “Minimal Attention” from the user.

Green Data Looks Good
Yellow Data Pay Attention
Red Data Pay Strict Attention

(A) Spamless - "Teleport" button helps you to reduce spam mails

(B) Better Privacy - Since "Green Data" is useless in the third party hands, it offer better privacy

(C) Transparency - You can clearly see what data you gave access to the website from your Combox page.

(D) Simplicity - The purpose of Teleport button is Signup and Login. Nothing more. No other data access other than the fields you see in the traditional signup, login and edit profile forms.

Because 99% of the time, when people click the “Signup with Google” or “Signup with Facebook” kind of button, they are there to SIGNUP. Not to give access to their entire data.

Note that we are not a social networking website and we have no plans to become one. So there is no need for a website to put message like “We don't share anything without your permission”. So our "Teleport" button is "less scary"
