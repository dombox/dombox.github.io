Solution Summary

Heads Up! Our solution may not make complete sense unless you view our presentation. So don't judge this project by only reading this summary. Consider this summary as a half-baked version.

We have two kind of mailboxes. (A) Normal Mailboxes (B) Isolated Mailboxes

We solve the email spam problem in three phases. Isolation, Restriction and Injection

Phase 1: Isolation

This phase deals with Isolated Mailboxes.

In this phase all the websites you signup are isolated. An isolated mailbox will be given to each website you signup. Only that particular website can send mails to that isolated mailbox by default.

However that particular website can whitelist certain third party domains in their DNS to send mail to their isolated mailbox. These whitelisted domains are called "Sender Alias Domains" in our system

e.g. An "Isolated Mailbox" is created for amazon.in. By default amazon.in can mail to that box. To accept mail from jeff@amazon.com, amazon.in should whitelist amazon.com in their DNS

All websites have exclusive unrestricted privilege to send mails to their "Isolated Mailbox".

A user can have unlimited "Isolated Mailboxes"

Note: An "Isolated Mailbox" is a property of both website owner and the mail user. Only website owner has write privileges and Only the mail user has read and delete privileges

Phase 2: Restriction

This phase deals with Normal Mailboxes.

A normal mailbox works exactly the same way just like your @gmail.com mailbox. However our normal mailboxes comes with an option called "Restricted Mode".

When enabled, it allows mails only from the contacts found in your "Contacts" list.

Since we already isolated website related mails with "Isolated Mailboxes", normal mailboxes contains only the real conversational mails you get from your friends, family or someone you already know. So when you enable "Restricted Mode", it's more like hanging a board with sign "Authorized Personnel Only"

Keep in mind, you cannot have "Restriction" without "Isolation". Because we offload all website related mails to "Isolated Mailboxes" and then keep only the conversational mails in "Normal Mailboxes"

Phase 3: Injection

Both "Isolation" and "Restriction" phase makes our system bullet proof from spammers. But it also makes the system bullet proof from Genuine senders who are not on your contact list. Let’s call them "Genuine Strangers".

This phase makes sure only the "Genuine Strangers" can able to mail you without any issues. But not the spammers. This phase contains two options. (1) Phone number validation (2) Attention Fee

In "Phone number validation", the sender need to enter your phone number correctly to allow mails.

In "Attention Fee", an amount will be charged from the sender. If you mark the received mail as genuine, then the contact will be added to your "Contacts" and the amount will be refunded to the sender. The amount can be set from 1 cent to $1000. Default value: 1 cent.

Technical Overview

Architecture

Our system contains four components. Layers, Filters, Scanners and Boxes

Component 1: Layers

Layer Purpose
Encryption Layer Confidentiality. i.e. To communicate securely
Authorization Layer To whitelist IP addresses for the "Sending Domain"
Alias Layer To allow third party domains to send mails to the isolated mailbox
Authentication Layer Authenticity & Integrity. i.e. To prevent message modification on transit
Alignment Layer To prevent email spoofing

Component 2: Filters

Filter Purpose
Spam Filter You know what it is. We use this for backward compatibility
Anomalies Filter Less aggressive spam filter. Primarily targets Phishing and Malware mails

Component 3: Scanners

Scanner Purpose
Malware Scanner To scan for viruses

Component 4: Boxes

Box Purpose
Primary (P) To have a "Normal Mailbox" that works exactly like Gmail
Mailbox (M) To use as a 3rd party Mail Client. e.g. @gmail.com &
To use as a 3rd party Mail Server. e.g. @yourcompany.com
Dombox (D) To let consumers have control over the "Isolated Mailbox"
Hybrid (H) To provide "One-Click" newsletter subscription service
Combox (C) To let businesses have control over the "Isolated Mailbox"

Note:

Primary (P) and Mailbox (M) box types are "Normal Mailboxes".

Dombox (D), Hybrid (H) and Combox (C) box types are "Isolated Mailboxes"

Spam Filter is applicable only for Primary (P), Mailbox (M) and Dombox (D) box types

Anomalies Filter is applicable only for Hybrid (H) and Combox (C) box types

Website Owners

If you are a website owner, please pay attention.

Our Parallel Internet is backward compatible with the current Internet. Meaning our mail users can use your website as usual.

However your website may not be forward compatible with our Parallel Internet. Our Internet gonna create a new problem for your business. "Unstable Users". i.e. Your users gonna disappear anytime

Isolated Mailboxes comes with two important options. "Make Offline" and "Delete". When a box is offline, mails won't be accepted to that box. When a box get deleted that means you lost a customer

We have a new box type called "Combox" to solve this problem. Our "Combox" box type revokes "Make Offline" and "Box Deletion" privileges from the consumers.

The term "Combox" refers to a Dombox that is under contract. In other words Combox refers to a "Contract-based Dombox".

Combox box type is over protective. If a mail doesn't pass those 5 layers, then mail will be immediately rejected

Combox box type available only for our "Portal Partners". A "Portal Partner" is a website that supports our "Teleport" button and sends only "Verified Mails i.e. 5 layer passed mails" to their "Isolated Mailbox". Combox can be created only via "Teleport" button

When your domain become our Portal Partner, we will disable other two isolated mailbox types. When they try to create an isolated mailbox for your domain, they will get a message like this.

"Example.com is our Portal Partner. Please use the Teleport button found in <this page> to create an Isolated Mailbox"

Via Combox, Website owners get "Stable Users", on the other hand Consumers get "Spam-free" Isolated Mailbox. So it's a win-win on both sides

Teleport is our "Contract-based Authentication Service".

Purpose: Quick Signup and Login, To create Isolated Mailbox on signup, To establish a contract, To provide better privacy.
